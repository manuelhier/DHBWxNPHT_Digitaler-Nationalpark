## Project Overview

DHBW student project: a chatbot for **Nationalpark Hohe Tauern** (Austria's largest Alpine national park). Architecture is AnythingLLM as a headless RAG backend, served via nginx with a custom static frontend. The AnythingLLM embed widget handles the chat UI; the frontend just loads it dynamically.

## Repository Structure

```
docker-compose.yml      # Orchestrates all three services
.env                    # Active config — gitignored, copy from .env.example
.env.example            # Full reference of all supported env vars

anythingllm/
  config/
    system-prompt-v1.md # Versioned system prompts (German)
  data/                 # Source documents for RAG ingestion (German)

setup/
  setup.sh              # Idempotent init: full setup on first run, skipped if workspace exists
  Dockerfile

frontend/
  index.html            # Static chat UI served by nginx
  nginx.prod.conf       # Production nginx config (HTTPS, subdomains)
  nginx.conf            # Dev nginx config (HTTP, localhost)
  config.json           # Generated by setup container at runtime — gitignored
```

## Running the Stack

```bash
docker compose up -d
docker logs npht-setup   # watch setup progress
```

Three containers start:

| Container | Role | Port |
|---|---|---|
| `npht-anythingllm` | RAG backend | 3001 |
| `npht-setup` | Idempotent init, exits when done | — |
| `npht-frontend` | nginx serving the frontend | 80 |

The setup container waits for AnythingLLM to be healthy, then checks if the workspace already exists:

- **First run (no workspace):** creates workspace `hohe-tauern`, applies system prompt from `anythingllm/config/system-prompt-v1.md`, uploads and embeds all documents from `anythingllm/data/`
- **Subsequent runs (workspace exists):** skips all of the above — embeddings, chat history, and settings persist

Either way, it writes the API key and workspace slug to `/run/config/` for nginx to pick up.

Frontend: `http://localhost`  
Admin UI: `http://localhost:3001`  
Swagger: `http://localhost:3001/api/docs`

Restart after `.env` changes:
```bash
docker compose restart
```

## Environment Configuration

Copy `.env.example` → `.env` and fill in:

| Key | Purpose |
|---|---|
| `AUTH_TOKEN` | AnythingLLM admin password |
| `JWT_SECRET` | Token signing — generate with `openssl rand -hex 32` |
| `GROQ_API_KEY` | LLM inference (free tier at console.groq.com) |
| `GROQ_MODEL_PREF` | Currently `llama-3.1-8b-instant` |
| `ANYTHINGLLM_WORKSPACE_NAME` | Workspace slug (default: `hohe-tauern`) |

Embedding runs natively inside the container (no extra API key). Vector DB is LanceDB (embedded, no separate service).

## How the Embed Widget Works

`frontend/index.html` fetches `/anythingllm-slug.json` at page load to get the workspace slug, then uses it to communicate with the AnythingLLM backend via the proxied `/api/` path.

nginx proxies:
- `/api/` → `http://anythingllm:3001/api/`
- `/embed/` → `http://anythingllm:3001/embed/` (widget script)

The widget color scheme uses park green (`#2f6b3f`) throughout.

## System Prompt

The chatbot answers in German by default, switches to English if the user writes in English. Prompts are versioned in `anythingllm/config/`. The active prompt is applied by `setup.sh` only on first run (when the workspace is created) — subsequent edits should be made via the AnythingLLM Admin UI.

Emergency number for mountain incidents: **140** (Bergrettung Österreich) — always referenced in the system prompt.

## Frontend

`frontend/index.html` is a static page replicating the hohetauern.at hero section (nav, Großglockner background, park stats). Chat opens from a hero CTA button or the AnythingLLM floating bubble (bottom-right). No build step required.

Planned production stack: React + Vite + Tailwind CSS + `@assistant-ui/react` + `openai` npm package.

## Chatbot Use Cases

**General Information** — park zones, visitor centres, history, three state sections (Salzburg, Tyrol, Carinthia)

**Accommodation & Travel** — alpine huts, base villages (Zell am See, Matrei, Heiligenblut), Postbus/train, parking

**Hiking & Activities** — trails by difficulty/distance/elevation, Tauern Höhenweg, via ferratas, winter activities

**Nature & Wildlife** — Steinbock, Bartgeier, Murmeltier, Steinadler; glaciers (Pasterze); Natura 2000 habitats

## RAG Document Scope

Documents in `anythingllm/data/` are curated manually from official park authorities — not scraped. Primary language is German. Current documents cover general park info, hotels, restaurants/huts, and hiking routes.
